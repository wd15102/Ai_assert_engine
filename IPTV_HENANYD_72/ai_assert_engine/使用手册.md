## 🚂 ChannelRunner 详细用法

### 实例化参数

```python
from lib.channel_runner import ChannelRunner

runner = ChannelRunner(
    channel_name="直播",            # 必填：中文频道名
    bind_instance_id="zhibo_6.0",   # 必填：测试桩 bindInstanceId
    output_dir=None,                # 选填：默认 auto/{channel_name}/
    screenshot_prefix=None,         # 选填：默认 channel_name 首二字
    checkpoint_key=None,            # 选填：默认 "{channel_name}频道"
    report_title=None,              # 选填：默认 "{channel_name}频道模板自动巡检"
)
runner.run()  # 一键执行
```

### 内部步骤

| 步骤 | 方法 | 说明 |
|------|------|------|
| 1 | `navigate()` | HOME → 频道列表sn差值 dpad → 到达 → dpad_down 进内容区 |
| 2 | `load_stub()` | 解析测试桩 JSON → 模板列表 |
| 3 | `load_checkpoints()` | 加载 checkpoints.json → expected_map |
| 4 | 逐模板检查 | 循环 `TemplateInspector.inspect()` → 截图/裁剪/AI判决 → 记录结果 |
| 5 | `_report_final()` | `generate_html_report()` + 终端摘要 |

### expected=false 模板

checkpoints.json 中设置 `"expected": false` 的模板不会被截图也不会调 AI。处理方式：XML dump → `extract_template_regions()` → 判断模板名是否出现。零截图零 AI 调用。

## 🔍 TemplateInspector 详细用法

```python
from lib.template_inspector import TemplateInspector

ins = TemplateInspector(output_dir='.', screenshot_prefix='tv')

# 1. 截图+焦点
snap = ins.screenshot_with_focus('page1')
# → {screenshot: '...png', marked: '...marked.png', focus_rect: [x,y,w,h]}

# 2. 模板区域裁剪
crop = ins.crop_template_region('首屏4横图通栏模板', snap['screenshot'],
                                 focus_rect=snap['focus_rect'])
# → {crop: '裁剪图.png', marked: '裁剪+绿框图.png'}

# 3. AI检查
result = ins.check_template_with_ai('首屏4横图通栏模板',
    crop.get('marked') or crop.get('crop'),
    public_checks=[{'名称':'排版正常','说明':'模板内各卡片/元素，无重叠、无错位、无跑飞'}],
    exclusive_checks=[])
# → {total:1, passed:1, failed:0, items:[{name, passed, detail}]}

# 4. 判断模板是否不存在（expected=false）
ins.check_template_absent('底部通栏')  # → True/False

# 5. 一步到位（截图+裁剪+AI）
result = ins.inspect('首屏4横图通栏模板',
    public_checks=[...],
    exclusive_checks=[...],
    template_idx=1)
```

### AI 图片选择优先级

裁剪+标记图 > 纯裁剪图 > 全屏焦点标记图 > 原始截图

## ⚙️ ADB 降级链（自动管理，无需人工干预）

`dump_uiautomator()` 自动按 D→A→C→E 顺序尝试：

| 策略 | 原理 | 触发条件 |
|------|------|---------|
| **D** | Appium HTTP API `page_source`（推荐，不 fork 不 OOM） | Appium 端口 4723+ 存活 |
| **A** | `adb shell uiautomator dump --compressed` | OOM 时 `--compressed` 减少内存 |
| **C** | 杀僵尸 uiautomator 进程 + 5 秒冷却 + 重试 A | A 失败后触发 |
| **E** | uiautomator2 Python 库 | C 也失败，最终兜底 |

`kill_stale_uiautomator()` 逐行解析 `ps` 输出，**跳过 Appium 相关进程**。

## 📊 Logcat 崩溃监控

```python
from lib.log_checker import LogcatWatcher
from lib.log_analyzer import LogAnalyzer

watcher = LogcatWatcher()
watcher.start(filters=['mangotv'])
# ... 执行操作 ...
events = watcher.stop()

# 用 AI 分析根因
if any(e['level'] == 'error' for e in events):
    report = LogAnalyzer().analyze(events)
    print("根因:", report['root_cause'])
```

### 过滤规则

| 来源 | 匹配条件 | 处理 |
|------|---------|------|
| AndroidRuntime FATAL EXCEPTION | 含 `CRASH_MONITOR_PKG` | → `error` |
| 首行无包名 | — | → `warn` 降级 |
| libc 含 "uiautomator" | Appium/测试框架 | → `ignore` |
| libc 不含 | — | → `warn` |
| 其他非目标包 | — | → `ignore` |
