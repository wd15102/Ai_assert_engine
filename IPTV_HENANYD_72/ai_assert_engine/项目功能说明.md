# AI驱动首页配置 + AI断言引擎

## 📌 做的是什么

> **测试桩推送到终端 → AI自动采集截图+裁剪模板 → AI视觉断言 → 生成HTML测试报告**

不再靠人眼盯电视屏幕。所有终端展示验证——模板对齐、焦点到位、文字不乱码、海报不裂——全部交给 AI 自动完成。

## 📦 核心模块详解

### 1. ChannelRunner（巡检引擎）

**文件：** `lib/channel_runner.py`，~540 行

**职责：** 管理整个巡检流程的生命周期。

```python
from lib.channel_runner import ChannelRunner

runner = ChannelRunner(
    channel_name="电视剧",                   # 中文频道名
    bind_instance_id="dianshiju_6.0",       # 测试桩 bindInstanceId
    output_dir=...,                         # 截图/裁剪图输出目录
    screenshot_prefix='dsj',                # 截图文件前缀
    checkpoint_key='电视剧频道',              # checkpoints.json 键名
)
runner.run()  # 一键执行：导航→测试桩→checkpoints→逐模板→报告
```

**内部步骤：**
1. **导航** `navigate()` — HOME → 频道列表sn差值 → dpad_left/right 到达 → dpad_down 进内容区
2. **测试桩** `load_stub()` — 从 JSON 测试桩解析模板列表
3. **checkpoints** `load_checkpoints()` — 加载 expected_map（哪些模板expected=false跳过）
4. **逐模板** — 循环调用 TemplateInspector 检查每个模板
5. **模板间导航** `navigate_to_next_template()` — 空间坐标比对，dpad_down逐步前进
6. **日志分析** — 捕获 log_checker 事件丢给 log_analyzer 做根因分析
7. **报告** `_report_final()` — HTML报告 + 终端摘要

**expected=false 处理：** `_build_expected_false_result()` — 委托 TemplateInspector.check_template_absent() 从 XML 区域列表判断模板是否不存在，零截图零 AI 调用。

**底部兜底裁剪：** `_bottom_crop_fallback()` — 最后一个模板且裁剪失败时，用 extract_template_regions 最后一区域的 Y 坐标做全宽裁剪。

**底部重复焦点检测：** 导航循环中，焦点 Y≥900 且相邻两次焦点边界框几乎相同（△x<10, △y<10），判定已滚动到底部。

### 2. TemplateInspector（单模板检查器）

**文件：** `lib/template_inspector.py`，~440 行

**职责：** 一个模板的完整检查流程，可独立于 ChannelRunner 使用。

```python
from lib.template_inspector import TemplateInspector

ins = TemplateInspector(output_dir='.', screenshot_prefix='tv')

# 方法1: 截图+XML+焦点+绿框
snap = ins.screenshot_with_focus('template_1')
# → {'screenshot':..., 'marked':..., 'focus_rect':[x,y,w,h], 'desc':'...'}

# 方法2: 模板区域裁剪
crop = ins.crop_template_region('首屏4横图通栏模板', snap['screenshot'],
                                 focus_rect=snap['focus_rect'])
# → {'crop':'裁剪图.png', 'marked':'裁剪+绿框图.png'}

# 方法3: AI视觉检查
result = ins.check_template_with_ai(
    '首屏4横图通栏模板',
    crop.get('marked') or crop.get('crop'),
    public_checks=[{'名称':'排版正常','说明':'模板内各卡片/元素，无重叠、无错位、无跑飞'}],
    exclusive_checks=[{'名称':'焦点位置','说明':'焦点是否在第一个海报上？'}],
)
# → {'template_name':..., 'total':N, 'passed':N, 'failed':N, 'items':[...]}

# 方法4: 判断模板是否不存在（expected=false）
absent = ins.check_template_absent('不支持模板名')
# → True（不存在，符合预期） / False（异常出现）

# 方法5: 一站式检查（上面4步合并）
result = ins.inspect('首屏4横图通栏模板',
                      public_checks=public_checks,
                      exclusive_checks=exclusive_checks,
                      template_idx=1)
# → 兼容 channel_runner 的结果 dict
```

**AI 判决三级降级：**
| 级别 | 方法 | 耗时 |
|------|------|------|
| L1 | JSON解析 `{"result":"pass"/"fail"}` | ~1ms |
| L2 | 正则提取JSON对象 | ~2ms |
| L3 | 关键词判定（"pass"在前60字且无"fail"） | ~1ms |

### 3. ai_service（AI 视觉服务 — 核心亮点）

**文件：** `lib/ai_service.py`，~1550 行

**职责：** 统一 AI 视觉问答入口，支持 VQA、OCR、焦点感知、全自动检查、多模型路由。

```python
from lib.ai_service import ai  # 模块级单例
```

**方法列表：**

| 方法 | 返回值 | 用途 |
|------|--------|------|
| `ai.vqa("问题")` | `bool` | 单条布尔断言判断 |
| `ai.ask("问题")` | `str` | 开放式提问 |
| `ai.vqa_batch(["问题1","问题2"])` | `[bool,bool]` | 批量断言 |
| `ai.vqa_object("问题", response_format=Schema)` | `BaseModel` | 结构化结果 |
| `ai.ask_text("问题", model='xxx')` | `str` | 纯文本推理（非视觉模型） |
| `ai.check_focus("模板名", card_index)` | `bool` | 焦点检测 |
| `ai.ocr()` | `[{'text':'xx','x':92,...}]` | 全部文字+坐标 |
| `ai.ocr_text()` | `['文字1','文字2']` | 纯文字列表 |
| `ai.ocr_exists("文字")` | `bool` | 判断有无指定文字 |
| `ai.ocr_find("文字")` | `{'text':...,'cx':...,'cy':...}` | 找文字坐标 |
| `ai.ocr_find_regex("模式")` | `{'text':...,'x':...,'y':...}` | 正则匹配第一个文字 |
| `ai.compare_images(a, b)` | `0.0~1.0` | 图片相似度 |
| `ai.is_similar(a, b, threshold=0.85)` | `bool` | 是否相似 |

**新增焦点感知方法（2026-07-16）：**

| 方法 | 用途 |
|------|------|
| `ai.ask_focus("当前焦点在哪个模板上？")` | 焦点框感知问答：先解析 XML 找到焦点位置和模板名，再发给AI |
| `ai.generate_checkpoints(image)` | 根据模板裁剪图，AI 分析布局自动生成检查项 |
| `ai.generate_checkpoints_for_focus(channel_hint="直播")` | 自动截图→找焦点→裁剪→生成检查项，全自动 |
| `ai.auto_check(image, channel_hint="直播")` | 一步到位：生成检查项→逐条检查→返回通过/失败统计 |
| `ai.load_context(builder, clips_dir)` | 加载模板上下文和裁剪目录，之后调用 `vqa_template/ask_template` 不需传图 |
| `ai.clear_context()` | 清除已加载的上下文 |
| `ai.ask_template("问题", "模板名")` | 使用模板专属提示词的 ask，自动匹配裁剪图 |
| `ai.vqa_template("问题", "模板名")` | 使用模板专属提示词的 vqa，自动匹配裁剪图 |
| `ai.ask_channel("问题", builder)` | 使用频道全量模板提示词 |
| `ai.check_template("模板名")` | 自动检查指定模板的公共检查项 + 专属检查项 |

**全自动检查示例：**
```python
# 一键：截图→找焦点→裁剪→生成检查→逐项检查→出结果
result = ai.auto_check(channel_hint="直播")
print(f"通过 {result['通过']}/{result['通过'] + result['失败']}")
```

### 4. adb_utils（ADB 硬件层）

**文件：** `lib/adb_utils.py`，~1200 行

**核心降级链** `dump_uiautomator()`：

```
D(Appium HTTP API) → A(shell --compressed) → C(杀僵尸+冷却重试A) → E(uiautomator2库)
```

| 策略 | 原理 | 失败条件 |
|------|------|---------|
| D | Appium page_source API（不fork，无OOM风险） | Appium未启动 |
| A | `adb shell uiautomator dump --compressed` | OOM/内核Kill |
| C | 杀残留进程+cooling+重试A | 系统资源耗尽 |
| E | uiautomator2 Python库 | 库未安装/import失败 |

```python
# 常用操作
screenshot('filename.png')           # → 路径
home(), back(), ok()                 # 单键
dpad_up/down/left/right()            # 方向键
keyevent('KEYCODE_MENU')             # 任意键
adb_shell(['cmd',...])               # ADB shell 命令
kill_stale_uiautomator()             # 杀残留 uiautomator（跳过 Appium 进程）
prewarm_appium()                     # 预热 Appium session
dump_uiautomator(out_dir='.', retries=2, time_budget=30)  # → XML 路径/None
wait_page_stable(timeout=5)          # 智能等待页面稳定
```

### 7. log_checker（实时Logcat监控）

**文件：** `lib/log_checker.py`，~480 行

**职责：** 巡检过程中后台线程实时捕获崩溃/ANR。

```python
from lib.log_checker import LogcatWatcher

watcher = LogcatWatcher()
watcher.start(filters=['mangotv', 'hylink'])
events = watcher.stop()
```

**包名过滤规则（三层过滤）：**

| 来源 | 规则 | 处理 |
|------|------|------|
| AndroidRuntime FATAL EXCEPTION | 含 `CRASH_MONITOR_PKG` | → `error` 级别报警 |
| AndroidRuntime 首行无包名 | — | → `warn` 降级 |
| 非目标包 | — | → `ignore` 忽略 |
| libc 含 "uiautomator" | Appium/测试框架自身 | → `ignore` 忽略 |
| libc 不含 "uiautomator" | — | → `warn` 降级 |
| Fatal signal / 其他 | 含目标包或空包名 | → `warn` 降级 |

## 🎯 断言模型

| 层级 | 手段 | 依赖 | 能发现什么 |
|------|------|------|-----------|
| 🟢 L1 接口层 | Charles 请求日志 + ADB | ADB | 接口通不通、图片404、资源加载 |
| 🟡 L2 日志层 | LogcatWatcher + LogAnalyzer | ADB + AI | 崩溃/ANR/空指针/JSON解析失败/根因分析 |


### 在 Python 内独立调用
```python
from lib.ai_service import ai

# 布尔判断
ai.vqa("当前焦点在首屏4横图通栏模板的第一个海报上吗？")

# 开放式提问
ai.ask("描述当前屏幕上所有可见模板")

# 全自动一键检查
result = ai.auto_check(channel_hint="直播")
print(result['通过'], result['失败'])

# 指定模型
ai.ask("描述一下", model="LongCat-2.0")
ai.ask_text("分析这段日志", model="agnes-2.5-flash")

# 独立检查一个模板
from lib.template_inspector import TemplateInspector
ins = TemplateInspector(output_dir='.')
result = ins.inspect('首屏4横图通栏模板', template_idx=1)
```

### 监控崩溃并分析根因
```python
from lib.log_checker import LogcatWatcher
from lib.log_analyzer import LogAnalyzer

watcher = LogcatWatcher()
watcher.start()
# ... 执行操作 ...
events = watcher.stop()
if any(e['level'] == 'error' for e in events):
    report = LogAnalyzer().analyze(events)
    print("根因:", report['root_cause'])
```

## 📋 checkpoints.json（2026-07-16 更新）

**文件：** `template_prompts/checkpoints.json`

公共检查项从 3 条扩展为 **5 条**：

| 名称 | 说明 |
|------|------|
| 排版正常 | 模板内各卡片/元素，无重叠、无错位、无跑飞 |
| 坑位海报正常 | 每个卡片的海报图正常显示，无黑图/灰图 |

## 📋 编码规范

见 `编码规范.md`（14条强制规则），AI 开发时必须遵守。

重点摘要：
1. **需求要先确认再写**，一次只问一个问题
2. **不得擅自加逻辑**，所有代码需求必须来自用户
3. **Python 3.9 兼容**，禁止 `dict | None`、`match/case` 等 3.10+ 语法
4. **测试桩不得修改**（只读）
5. **模板标题来源测试桩** `datas[*].title`，禁止硬编码
6. **修改后三段自检**：语法检查 → 编码检查 → 构建验证
